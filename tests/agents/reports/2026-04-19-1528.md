### Orchestrator notes — investigation leads for the cert-CRUD P0

The subagent's evidence narrows the scope:

1. **Reads work.** GET `/dashboard/certifications` renders correctly → middleware-level auth is fine for read requests.
2. **Other server actions worked this session.** Journey 04 ran ~9 minutes earlier at 1519 and successfully persisted the 8 imported employees + certs via the import commit server action. So the issue is **specific to the cert CRUD code path**, not a global session-broken state.
3. **The POST carries no cookie.** Playwright logged `cookie header length: 0` on the outgoing POST. Two shapes this can take:
   - (a) Client-side: the cert form uses a form action that routes through a different origin / different path that doesn't include cookies (e.g. a server-action endpoint on `/_next/` that doesn't match the cookie scope).
   - (b) Middleware: the server action's `createServerClient` instance in `cookies().setAll()` rewrites the auth cookie as expired, which Supabase does on "refresh token failure" paths.
4. **Server deliberately clears the cookie.** `Set-Cookie: ...=; Max-Age=0` means the server side *decided* to log the user out. This is not a passive missing-cookie case; the server handler ran and judged the session invalid.
5. **Likely investigation starting points** (in order of probability):
   - Inspect `src/app/dashboard/certifications/{new,[id]/edit}/actions.ts` (or wherever the cert CRUD server actions live) for how they call `createServerClient` and whether `cookies().set` is being invoked in a way that zeroes the token.
   - Compare with `src/app/dashboard/import/actions.ts` which works in this same session.
   - Check Next.js 15 server-action cookie forwarding — recent Next.js versions require explicit opt-in for cookie propagation on multipart form submissions.
   - Verify the middleware's matcher pattern doesn't exclude the server-action route.

**This P0 is almost certainly NOT related to the earlier ON CONFLICT P0** (that was a schema-drift issue on staging, already fixed in commit 6566dc6). This one is either application-logic or Next.js-config-level.
