## Actionable backlog — ranked

### Tier 1 — P0, unshipped work-blockers

1. **[Import] Non-xlsx file silently accepted** (Run 5 / Journey 04). Client validates only file extension; server-action error path doesn't reach the user. User uploads a garbage file, app shows review step with 0 rows, "Import 0 employees" button enabled. Fix scope: add content-type check after `XLSX.read`; on throw, route back to upload step with a red banner.
2. **[Import] Empty xlsx silently advances to review with auto-creation of cert type** (Run 5). Same failure mode: the review step is used as a general "post-upload landing" rather than a conditional "we parsed rows, please confirm". Fix scope: short-circuit to the upload step with a friendly "הקובץ ריק" message when parsed-row-count = 0; don't auto-mint a cert type from the sheet name alone.
3. **[Import] Unknown cert-type name auto-created without confirmation** (Run 5). Writing `NOPE_UNKNOWN_TYPE` in the Hebrew cert-type column silently creates a new first-class cert type owned by the manager. Fix scope: in the review step's `סוגי הסמכות שייווצרו` list, visually distinguish "already exists" from "new from your data"; require a "confirm create" checkbox for the latter.
4. ~~**[Cert CRUD] Server action logs admin out**~~ (Run 6) — **RECLASSIFIED AS NOISE.** Human-browser spot-check on 2026-04-19 evening: cert saves and returns to the cert list normally. The failure was a Playwright-specific cookie-handling quirk (server-action multipart POSTs not carrying `sb-auth-token` when driven through `page.click()`), not an app bug. Journey 05 needs a revised subagent prompt (e.g. use `page.evaluate(() => fetch(...))` with explicit credentials, or navigate-and-submit via URL) before the CRUD checks can be exercised.
5. **[Import, already fixed]** ON CONFLICT schema drift — cleared in commit `6566dc6` via self-healing seed.sql. Kept in table for audit trail only.

### Tier 2 — P1, silent data-loss bugs

6. **[Import] Invalid date string silently dropped** (Run 5). User types `תוקף תעודה = "not a date"`; row imports with all three date columns blank and status `לא ידוע`. The original typed value is never surfaced. Fix scope: during parse, if a date cell is non-null and non-parseable, push the row to `שורות שדולגו` with reason `תאריך תוקף לא תקין`, don't silently strip.
7. **[Import] `שורות שדולגו (N)` panel is incomplete** (Run 5). The skipped-rows summary only reports bad Israeli IDs — it hides rows with invalid dates, rows accepted under "unknown cert type" auto-create, and rows with entirely empty date fields. The count and reasons misrepresent what actually happened. Fix scope: every rejected-or-compromised row must appear in that panel with a specific reason.

### Tier 3 — P2, polish / sanity checks

8. **[Mobile] Tap-target audit** (Run 4, 6 findings). Buttons/links across `/dashboard/employees`, `/dashboard/certifications`, and employee detail measure 20-38px tall where the target is 40+. Single systemic fix: audit primary button/link primitives and add a `min-h-[44px]` token (plus possibly `touch-manipulation`) globally.
9. **[Import] Far-future date accepted without warning** (Run 5). `01/01/2099` passes validation. Fix scope: during parse, if a date is more than 10 years in the future, flag the row inline in review and require user confirm before commit.
10. **[Import] Name field accepts SQL-injection-style strings** (Run 5). `Robert'); DROP TABLE employees;--` renders verbatim. DB is safe (parameterized queries), but no name-shape validator exists. Fix scope: character-class whitelist on first-name / last-name columns in the parser.
11. **[Import] Cert with all dates empty silently created** (Run 5). Cert of status `לא ידוע` is unrecoverable — reminders can never fire for it. Fix scope: in the review step, flag rows where cert dates are all-empty with a "skip?" prompt.
12. **[Certifications] `לא ידוע` badge too understated** (Run 4). Missing-data certs blend with valid-data certs. Fix scope: color or emphasis distinction.

### Tier 4 — P3, cosmetic

13. **[Mobile Nav] Double-tap current tab doesn't scroll to top** (Run 1). Standard iOS pattern not implemented.
14. **[Mobile] Employees filter drawer collapsible** (Run 4). On 375px the first card is below the fold because three filter rows eat the top. Collapsing filters behind a trigger would help.

## Shared root causes observed

- **Tier 1 items 1-3 all stem from a common shape:** the import wizard's review step is treated as a general landing page after file upload, not as a conditional "we parsed valid rows, please confirm" gate. A single architectural pass — make the review step require `parsedRows.length > 0` AND `unknownCertTypes.length === 0 || userConfirmedUnknowns` — would collapse all three findings into one structural fix.
- **Tier 2 items 6-7 also share a root cause:** the import parser has two code paths: "hard skip with reason" (rejects) and "soft fill with null" (accepts under duress). The soft path drops information silently. Moving the soft-path cases into the hard-skip reporting surface (same `שורות שדולגו` panel with additional reasons) would fix both with one change.
- **Tier 3 item 8** is a design-system token fix, not per-component work.

**Estimated scope:** items 1-3 ≈ 1 PR, items 6-7 ≈ 1 PR, item 4 ≈ 1 PR after investigation, item 8 ≈ 1 PR, the rest ≈ 1 PR of small fixes. Five focused PRs would retire the entire actionable backlog.
